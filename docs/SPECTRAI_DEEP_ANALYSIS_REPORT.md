## 1. OutputParser - 输出解析引擎

### SpectrAI 实现分析
SpectrAI 的 `OutputParser.ts` 是一个完整的输出解析引擎，具有以下核心特性：

- **多层解析规则**：内置多种活动事件类型（tool_call、file_edit、command_exec、error、confirmation 等）
- **智能去重机制**：基于会话 ID 和事件类型的时间窗口去重，避免重复通知
- **Provider 自适应**：根据不同 AI Provider 的输出格式调整解析策略
- **Token 用量估算**：实时估算 Token 消耗并持久化到数据库
- **结构化读取器集成**：支持从 JSONL 格式准确检测错误事件
- **自定义规则支持**：通过 `~/.claudeops/custom-rules.json` 扩展解析规则

### ClawTeam 现状差距
当前 ClawTeam 的 parser 目录仅包含基础类型定义（`types.py`），缺少实际的解析逻辑实现。

### 升级建议

#### 实现方案
1. **创建 `parser/engine.py`**：实现核心解析引擎
2. **定义解析规则**：为不同活动类型创建正则表达式规则
3. **实现去重缓存**：使用时间窗口机制避免重复事件
4. **集成 Token 估算**：基于字符数估算 Token 消耗
5. **支持自定义规则**：通过配置文件扩展解析能力

#### 优先级：🔴 高优先级
- **依赖关系**：需要先实现基本的活动事件类型定义
- **预计工作量**：3-5 天
- **价值**：提供统一的输出解析能力，为后续通知、监控等功能奠定基础

## 4. ConfirmationDetector - 确认请求检测器

SpectrAI 的 `ConfirmationDetector.ts` 专门用于检测 AI 输出中的确认请求：
